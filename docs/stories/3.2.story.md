### Tech Stack

**CSS Framework:**
- **TailwindCSS**: Version 4.x (latest beta) - Utility-first CSS framework
- **@tailwindcss/vite**: Official Vite plugin for Tailwind processing
- **Rationale**: Tailwind 4 offers improved performance, native CSS cascade layers, and better Vite integration
- [Source: docs/architecture/tech-stack.md#CSS Framework]

**UI Component Library:**
- **shadcn/ui**: Component collection built with Radix UI primitives and styled with TailwindCSS
- **Not a package**: Components copied into project (`src/components/ui/`) for full customization
- **Dependencies**: `class-variance-authority` (component variants), `clsx` (conditional classes), `tailwind-merge` (merge Tailwind classes without conflicts)
- **Rationale**: shadcn/ui provides production-quality components without vendor lock-in, allowing modification for educational demonstration
- [Source: docs/architecture/tech-stack.md#UI Components]

**Theme System:**
- **CSS Variables**: HSL color values defined for light and dark modes
- **Dark Mode Strategy**: TailwindCSS `class` strategy (add `.dark` class to root element)
- **Base Colors**: Slate color palette for neutral theme foundation
- **Rationale**: CSS variables enable runtime theme switching without CSS recompilation (Story 3.4 requirement)
- [Source: docs/architecture/tech-stack.md#Theme Configuration]

**Path Aliases:**
- **@/components**: Maps to `src/components` for component imports
- **@/lib/utils**: Maps to `src/lib/utils` for utility functions
- **Configuration**: TypeScript `paths` in tsconfig.json + Vite `resolve.alias` in vite.config.ts
- **Rationale**: Cleaner imports (`@/components/ui/button` vs `../../components/ui/button`), easier refactoring
- [Source: docs/architecture/frontend-architecture.md#Import Strategy]

**components.json Structure:**


**Configuration Explanation:**

- **prefix: ""**: No prefix for Tailwind classes (use `bg-primary`, not `tw-bg-primary`)
- **aliases**: Match TypeScript path aliases for component imports

**Rationale**: Configuration optimized for Vite + React SPA (non-Next.js) with TypeScript, CSS variable theming, and clean imports.

### Theme Configuration (AC 4, 5)

**CSS Variables Strategy:**

```css
/* Light mode (default) */
:root {
  --background: 0 0% 100%;        /* White background */
  --foreground: 222.2 84% 4.9%;   /* Dark text */
  --primary: 222.2 47.4% 11.2%;   /* Dark blue primary */
  /* ... other variables */
}

/* Dark mode (when .dark class applied to <html>) */
.dark {
  --background: 222.2 84% 4.9%;   /* Dark background */
  --foreground: 210 40% 98%;      /* Light text */
  --primary: 210 40% 98%;         /* Light blue primary */
  /* ... other variables */
}
```

**TailwindCSS Integration:**

```javascript
// tailwind.config.js
export default {
  darkMode: ["class"],  // Enable dark mode via .dark class
  theme: {
    extend: {
      colors: {
        background: "hsl(var(--background))",  // Reference CSS variable
        primary: {
          DEFAULT: "hsl(var(--primary))",
          foreground: "hsl(var(--primary-foreground))",
        },
        // ... other colors
      },
    },
  },
}
```

**How It Works:**
1. CSS variables defined in `globals.css` with HSL values (no `hsl()` wrapper)
2. Tailwind config references variables with `hsl(var(--variable-name))`
3. Dark mode toggles `.dark` class on root element (Story 3.4)
4. All Tailwind classes (`bg-background`, `text-primary`) automatically switch themes

**Rationale**: HSL format allows opacity modifiers (`bg-primary/50` for 50% opacity), CSS variables enable runtime theme switching without CSS recompilation, class-based dark mode works in non-Next.js apps.

### Coding Standards

**Critical Rules for This Story:**
- âœ… **Component File Naming**: kebab-case for UI components (e.g., `scroll-area.tsx`, not `ScrollArea.tsx`)
- âœ… **Component Export Naming**: PascalCase for React components (e.g., `export const Button`)
- âœ… **CSS Variables**: Use HSL format without `hsl()` wrapper (e.g., `0 0% 100%`, not `hsl(0, 0%, 100%)`)
- âœ… **Import Aliases**: Use `@/` prefix for all component/utility imports
- âœ… **Tailwind Classes**: Use semantic theme classes (`bg-background`, `text-foreground`) over hardcoded colors (`bg-white`, `text-black`)
- âœ… **Class Merging**: Always use `cn()` utility when conditionally applying classes to support variant overrides

**Naming Conventions:**

| Element | Convention | Example |
|---------|-----------|---------|
| UI Component Files | kebab-case | `button.tsx`, `scroll-area.tsx` |
| Component Exports | PascalCase | `export const Button`, `export const ScrollArea` |
| Utility Functions | camelCase | `cn()`, `formatDate()` |
| CSS Variables | kebab-case with prefix | `--background`, `--primary-foreground` |
| Tailwind Classes | Semantic names | `bg-background`, `text-primary` |

**Class Utility Pattern:**

```tsx
import { cn } from "@/lib/utils"

// CORRECT: Allows variant overrides
<Button className={cn("w-full", className)}>Click me</Button>

// INCORRECT: Hardcoded classes cannot be overridden
<Button className="w-full">Click me</Button>
```
