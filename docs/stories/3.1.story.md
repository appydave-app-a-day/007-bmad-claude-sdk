### Tech Stack

**Frontend Framework:**
- **React**: Version 19.x (latest stable) - Component-based UI framework
- **Vite**: Version 7.x - Fast build tool with HMR, esbuild-powered bundling
- **TypeScript**: Version 5.7+ - Type-safe development with React JSX support
- **Rationale**: React 19 provides modern features (concurrent rendering, automatic batching), Vite 7 offers fastest HMR and build times
- [Source: docs/architecture/tech-stack.md#Frontend Framework, docs/architecture/tech-stack.md#Build Tool]

**Development Workflow:**
- **Development mode**: Vite dev server on port 5173 with HMR for component iteration
- **Production mode**: Express serves compiled bundle from `packages/client/dist` at `/chat` route
- **Concurrent execution**: `concurrently` package runs both Vite and Express during development
- **Rationale**: Dual-mode allows fast development (HMR) and realistic production testing (Express)
- [Source: docs/architecture/development-workflow.md#Development Mode]

**NPM Workspace Configuration:**
- **Monorepo structure**: NPM workspaces with `packages/client`, `packages/server`, `packages/shared`
- **Package naming**: Scoped packages with `@bmad-app/` prefix for consistency
- **Workspace scripts**: Root package.json scripts use `-w` flag to target specific workspaces
- **Cross-package dependencies**: Client imports shared types via `@bmad-app/shared` workspace dependency
- [Source: docs/architecture/unified-project-structure.md#Monorepo Structure]

### Express Server Integration (AC 6, 9)

**Chat Route Handler:**

```typescript
// packages/server/src/server.ts

import express from 'express';
import path from 'path';

const app = express();

// Development mode: Vite dev server runs on port 5173 with HMR
// Production mode: Express serves compiled React bundle from /chat route

// Serve React app at /chat route
app.use('/chat', express.static(path.join(__dirname, '../../client/dist')));
app.get('/chat', (req, res) => {
  res.sendFile(path.join(__dirname, '../../client/dist/index.html'));
});

// Existing routes (from previous stories)
app.use(express.static(path.join(process.cwd(), 'public'))); // Serve generated HTML from /public

// ... rest of server code
```

**Key Implementation Details:**
- `/chat` route serves React SPA from compiled bundle
- Root routes (`/`, `/products`, etc.) continue to serve generated HTML from `/public` directory
- Path resolution: `../../client/dist` relative to compiled server file location (`packages/server/dist/server.js`)
- Express static middleware serves bundled assets (CSS, JS) from `/chat` directory
