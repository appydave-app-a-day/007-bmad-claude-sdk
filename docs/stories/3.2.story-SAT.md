## Troubleshooting

### Issue: Vite dev server won't start (port 5173 in use)
**Symptom**: `Error: Port 5173 is already in use`
**Solution**:
1. Kill existing Vite process: `lsof -ti:5173 | xargs kill -9`
2. Or change port in `vite.config.ts`: `server: { port: 5174 }`

### Issue: Build fails with "Cannot find module '@/components/ui/button'"
**Symptom**: TypeScript error during build
**Solution**:
1. Verify `tsconfig.app.json` includes `"@/*": ["./src/*"]` in paths
2. Verify `vite.config.ts` includes `alias: { "@": path.resolve(__dirname, "./src") }`
3. Run `npm install` to ensure dependencies installed

### Issue: Styles not applying in browser
**Symptom**: Components render but look unstyled (no colors, no spacing)
**Solution**:
1. Verify `globals.css` imported in `src/main.tsx`: `import './globals.css'`
2. Check browser console for CSS loading errors
3. Hard refresh browser (Ctrl+Shift+R) to clear cache
4. Verify Tailwind plugin in Vite config: `plugins: [react(), tailwindcss()]`

### Issue: Button variants all look the same
**Symptom**: Secondary, outline, destructive buttons identical to default
**Solution**:
1. Inspect button element in DevTools
2. Verify `variant` prop passed correctly: `<Button variant="secondary">`
3. Check `button.tsx` uses `class-variance-authority` for variants
4. Reinstall button component: `npx shadcn@latest add button`

### Issue: CSS variables show as `var(--background)` instead of actual color
**Symptom**: DevTools Computed tab shows variable name, not HSL value
**Solution**:
1. This is normal - click the variable to see resolved value
2. If resolved value is `undefined`, check `globals.css` defines the variable
3. Verify Tailwind config references variables with `hsl(var(--variable-name))`

### Issue: Production build loads but shows blank page
**Symptom**: `http://localhost:3000/chat` loads but displays nothing
**Solution**:
1. Check Express server serves from `/chat` route
2. Verify Vite config has `base: '/chat/'`
3. Check browser console for 404 errors on assets
4. Rebuild client: `npm run build -w @bmad-app/client`
5. Restart Express server

### Issue: Dark mode variables not found in DevTools
**Symptom**: Only `:root` variables visible, no `.dark` class
**Solution**:
1. This is expected - `.dark` variables only visible when class applied to `<html>`
2. To test: In DevTools Elements, add `dark` class to `<html>` element manually
3. Full dark mode toggle implemented in Story 3.4
